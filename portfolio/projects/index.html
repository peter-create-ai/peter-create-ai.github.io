OmniMerchant — Multilingual AI Customer Service Agent

Role: Full-Stack Lead Developer · Timeline: Apr 2026 – Jun 2026
GitHub: github.com/RyanCoreAI/spring-ai-crossborder-customer-service

The Problem

Cross-border e-commerce stores receive 200–500 multilingual inquiries daily (English, Chinese, Spanish, French, Arabic, etc.). Manual support costs $20K–$50K/month in staffing, and response times spike during off-hours due to timezone gaps. Small-to-medium sellers cannot afford 24/7 human agents.

The Solution

OmniMerchant is a 7×24 multilingual AI customer service platform built on Spring AI, designed for Shopify, Amazon, and TikTok Shop sellers. It handles ~70% of tier-1 support workload, saving an estimated $3K–$7K per store per month.

Key Technical Achievements

1. Reliable Tool Calling (99.4% Success Rate)

Implemented 6 business tools via Spring AI Tool Calling — order lookup, logistics tracking, refund-policy RAG, product search, multilingual translation, and human escalation. Built an AOP-based execution layer with parameter validation, hallucination guardrails via result verification, and a three-tier fallback mechanism.

2. Cost-Optimized Multi-Model Routing (60% Token Cost Reduction)

Designed a confidence-driven dynamic router across GPT-4o-mini, Claude Haiku, and DeepSeek models, combined with English-pivot translation and intent caching strategies. Achieved P99 latency < 3s while reducing token costs by over 60%.

3. Hybrid RAG with 93% Recall@3

Built a dual-knowledge-base RAG system (policy documents + product catalog) using PGVector with HNSW indexing, BM25 sparse retrieval, RRF fusion, and BGE Cross-Encoder reranking. Conducted 4 ablation studies, improving Recall@3 from 78% to 93% on a custom benchmark.

4. Automated Evaluation Pipeline

Created a 200-sample multilingual intent test set and 100-sample RAG test set. Built automated evaluators (Precision, Recall, F1, MRR, Faithfulness) for continuous regression testing. Iterated on bad cases to improve intent accuracy from 91.5% to 96.5%.

5. Production-Grade Infrastructure

Implemented tenant-level token billing + three-layer rate limiting (QPS, monthly budget, model concurrency) using Redis Lua scripts, with budget 80% alert and 95% auto-degradation. Added OpenTelemetry tracing, Prometheus 30+ business metrics, ELK centralized logging, and 5 Grafana production dashboards — reducing incident response from hours to minutes.
